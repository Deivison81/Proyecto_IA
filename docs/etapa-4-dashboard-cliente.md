# Etapa 4 - Dashboard Cliente

## Objetivo cumplido
Se implemento un dashboard funcional para el rol Cliente con creacion de solicitudes, seguimiento de estado e historial persistente en modo mock.

## Funcionalidades entregadas
1. Resumen con metricas rapidas de solicitudes.
2. Formulario para crear nuevas solicitudes.
3. Seleccion de servicio y prioridad.
4. Historial filtrable por estado.
5. Vista de detalle por ticket.
6. Timeline basico de seguimiento.
7. Persistencia por usuario en localStorage.

## Alcance actual
1. Datos mock por cliente autenticado.
2. No hay integracion backend aun.
3. Los tickets creados quedan disponibles al recargar el navegador para el mismo usuario.
